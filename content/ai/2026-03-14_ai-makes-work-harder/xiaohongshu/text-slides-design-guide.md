# 小红书长文拆分为多图文字卡片 — 设计规范

## 一、每张图放多少文字

- **每张文字卡片控制在100-200字**（中文字符），确保用户滑动浏览时每页3-5秒可读完
- 以本篇内容（约550字正文）为例，建议拆成 **4-5张内容页 + 1张封面 + 1张结尾页**，共6-7张图
- 每张图最多 **3-5个短段落**，每段不超过2-3行
- 小红书最多支持9张图，建议总量控制在 **6-9张**

### 本篇内容拆分建议

| 页码 | 内容 | 预估字数 |
|------|------|----------|
| 第1页 | 封面：标题「有了AI后，上班更累了？」| 标题文字 |
| 第2页 | 引入：效率高了但更累了 + 原因一（同时做的事变多） | ~120字 |
| 第3页 | 原因二（AI产出要review）+ 原因三（期望值高了） | ~120字 |
| 第4页 | 原因四（决策疲劳）+ 核心总结（AI解放执行力不解放注意力） | ~130字 |
| 第5页 | 解决方案：workflow化 + 具体例子 | ~130字 |
| 第6页 | 结尾金句 + 互动引导 + 话题标签 | ~50字 |

## 二、字体排版参数（基于1080x1440画布）

### 字号规范

| 元素 | 字号（px） | 字重 | 说明 |
|------|-----------|------|------|
| 页面大标题 | 60-72px | Bold/粗体 | 每页顶部的小标题或关键词 |
| 正文内容 | 36-42px | Regular | 主要阅读文字，不能小于32px |
| 强调文字 | 42-48px | Bold | 关键句、金句加粗突出 |
| 页码/装饰文字 | 24-28px | Light | 角落页码、来源标注等 |
| 封面主标题 | 80-100px | Heavy/特粗 | 封面视觉焦点 |

### 间距规范

| 参数 | 数值 | 说明 |
|------|------|------|
| 行间距 | 字号的 1.6-1.8 倍 | 例如36px字号配58-65px行高 |
| 段间距 | 字号的 2.5-3 倍 | 段落间留出明显呼吸空间 |
| 左右边距 | 80-120px | 画布两侧留白，防止文字贴边 |
| 上下边距 | 100-160px | 上方留更多空间给标题区域 |
| 文字区域安全区 | 距边缘至少 80px | 避免手机显示时被裁切 |

### 字体推荐

- **标题**：思源黑体 Bold / 阿里巴巴普惠体 Bold / 站酷快乐体
- **正文**：思源黑体 Regular / 苹方 Regular / 鸿蒙字体
- 全篇统一使用 **1-2种字体**，不要超过3种

## 三、风格一致性

**结论：所有页面必须保持统一的视觉风格。**

具体要求：
- **配色方案统一**：选定2-3个主色，所有页面使用相同的色彩组合
- **排版结构统一**：标题位置、正文区域、装饰元素位置每页保持一致
- **字体搭配统一**：全篇只用1-2种字体，不要每页换字体
- **背景风格统一**：要么全部纯色背景，要么全部纹理背景，不要混搭
- **唯一例外**：封面页和结尾页可以在统一风格基础上做适当变化（如放大标题、增加图形）

**为什么要统一？**
- 小红书算法对调性一致的内容识别更快，有利于分发推荐
- 用户滑动浏览时视觉连贯，阅读体验更好
- 提升账号专业度和品牌辨识度

## 四、让文字页不无聊的视觉技巧

### 必用元素

1. **色块分区**：用浅色圆角矩形色块作为文字背景，将不同段落/要点区分开
2. **编号标记**：用彩色圆形数字编号（如 ❶❷❸）标记要点，比纯文字层次清晰
3. **关键句高亮**：重要句子用色块底色或加粗+变色处理
4. **小图标/emoji**：每个要点前加一个相关图标，增加视觉锚点
5. **分隔线**：段落间用细线或装饰性分隔线（虚线、波浪线）

### 进阶技巧

- **渐变色块**：标题区域用微渐变背景，增加质感
- **圆角卡片嵌套**：正文放在圆角卡片里，卡片外是装饰背景色
- **留白呼吸感**：文字不要铺满，至少留30-40%的空白区域
- **装饰性图形**：角落放轻量几何图形（圆点、线条、小色块）
- **引号装饰**：金句用大号引号「」或装饰性引号包裹

### 避免的错误

- 全页密密麻麻的纯文字（无分段、无色块）
- 背景图太花导致文字看不清
- 颜色超过4种导致视觉混乱
- 装饰元素过多抢走文字焦点

## 五、图片尺寸

**所有页面统一使用 3:4 竖版比例。**

| 参数 | 推荐值 |
|------|--------|
| 比例 | 3:4（竖版） |
| 推荐像素 | 1080 x 1440 px |
| 高清像素 | 1242 x 1660 px |
| 最低像素 | 900 x 1200 px |
| 文件格式 | PNG（文字清晰度最佳）或 JPG（照片类） |
| 文件大小 | 单张不超过 10MB |

**为什么选3:4？**
- 竖版封面在信息流中显示面积比横版大约40%，曝光率更高
- 小红书同一篇笔记只支持一种比例，不能混搭
- 3:4在手机上几乎全屏显示，文字可读性最佳

## 六、常见排版版式

### 版式A：标题 + 正文段落

```
┌──────────────────┐
│     [小标题]      │  ← 大号加粗，居中或左对齐
│                    │
│  正文段落第一段    │  ← 常规字号
│  ……              │
│                    │
│  正文段落第二段    │
│  ……              │
│                    │
│          [页码]    │
└──────────────────┘
```
**适用场景**：叙述性内容、故事型段落

### 版式B：编号要点

```
┌──────────────────┐
│     [小标题]      │
│                    │
│  ❶ 要点一标题      │  ← 加粗
│     要点一说明      │  ← 常规，缩进或用色块
│                    │
│  ❷ 要点二标题      │
│     要点二说明      │
│                    │
│  ❸ 要点三标题      │
│     要点三说明      │
└──────────────────┘
```
**适用场景**：干货总结、原因分析、步骤教程（本篇的"四个原因"适合用这个）

### 版式C：金句/引用

```
┌──────────────────┐
│                    │
│                    │
│   「 金句内容      │  ← 大号字体，居中
│     金句第二行 」  │
│                    │
│    —— 出处/总结   │  ← 小号，右对齐
│                    │
│                    │
└──────────────────┘
```
**适用场景**：核心观点页、结尾总结页（本篇的"AI解放执行力但没解放注意力"适合用这个）

### 版式D：对比/前后

```
┌──────────────────┐
│     [小标题]      │
│                    │
│ ┌──────────────┐ │
│ │  以前…        │ │  ← 色块A
│ │  xxxxxx       │ │
│ └──────────────┘ │
│        ↓          │
│ ┌──────────────┐ │
│ │  现在…        │ │  ← 色块B（不同颜色）
│ │  xxxxxx       │ │
│ └──────────────┘ │
└──────────────────┘
```
**适用场景**：变化对比、痛点vs解决方案

### 版式E：结尾互动页

```
┌──────────────────┐
│                    │
│   核心金句总结     │  ← 大号字
│                    │
│   互动提问         │  ← 如"你们有类似感觉吗？"
│                    │
│   #话题标签        │  ← 小号，装饰性排列
│   #话题标签        │
│                    │
│   @账号名          │
└──────────────────┘
```
