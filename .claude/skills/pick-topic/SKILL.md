# 选题输入层（发内容前的第一步）

产出 = 3 张候选卡 → 用户选 1 → 讨论收窄 → 定稿一句话核心观点 → 写入 topics/*.md → 交给 /new-topic。

## 第一步：并行扫 4 路信号（都要真扫，不许凭记忆编）

1. **AI 圈时事**（WebSearch，中英双语，限定最近 7-14 天）：
   - 本周发布/融资/裁员/政策/爆款争论；"AI 新闻 本周"；"AI industry news this week"
   - 重点抓"情绪事件"：让打工人焦虑/兴奋/不服的事，比纯技术新闻值钱
2. **创作者生态**（WebSearch + agent-reach）：
   - 知识区/科技区最近什么话题和形式在涨（新榜、36氪、卡思数据）
   - agent-reach 可用通道：B站搜索（bili API）、V2EX 热帖、HN（exa 搜索）、任意网页（jina）
   - 小红书：`opencli xiaohongshu search <关键词>`（需 Chrome 登录态）
3. **自己账号数据**：
   - `opencli xiaohongshu creator-notes-summary`（近期各帖数据；登录态失效就跳过并注明）
   - memory 的 post_history 规律（哪类题历史表现好）
4. **存量池**：`topics/*.md` 待写清单——检查有没有老选题因为时事"突然到时候了"（存量+时事的组合往往是最强候选）

## 第二步：五维评分（1-5 分，校准锚点=电力篇）

| 维度 | 5 分的样子（电力篇为锚） |
|---|---|
| 时效热度 | 正蹭住一个本周仍在发酵的情绪事件（限token/ROI焦虑） |
| 账号契合 | AI 观察者人设 + 美国从业者信息差能放大它 |
| 比喻潜力 | 存在"一图胜千言"的画面级比喻（电动机装旧轴） |
| 事实可得 | 能 research 出硬数字和戏剧性细节（30年空窗、$40亿破产） |
| 差异化 | 中文圈还没人把这个角度讲透 |

淘汰线：任何一维 ≤2 分的候选不进 top3（宁缺毋滥，补扫信号）。

## 第三步：top3 候选卡（每张必须齐 7 项）

```
### 候选 N：<选题一句话>
- 为什么是现在：<绑定的具体时事/信号，带来源链接>
- 切入角度：<账号人设视角的独特切口>
- 预期钩子：<开头 5 秒/第一行的候选写法>
- 金句方向：<1-2 句雏形>
- 竞争情况：<谁写过、我们怎么差异化>
- 五维评分：时效X 契合X 比喻X 事实X 差异X = 总分
- 适合形态：图文 / 图文+视频 / 全家桶
```

## 第四步：用户选 → 讨论磨尖（强制，不许跳过）
- 历史规律：用户总会把泛题磨成更尖的观点（"谁玩得起AI"→"公司在踩刹车"）
- 用户选定后，主动提出 2-3 个收窄方向让用户挑/改
- 定稿标准：能用一句话说清"这篇真正想说什么"+ 一个明确的目标情绪

## 第五步：落账
- 定稿写入对应 topics/*.md 待写区（含一句话观点 + 绑定的时事引用）
- 提示用户可无缝接 `/new-topic`

## 纪律
- 信号必须真扫（带 Sources 链接），禁止用训练记忆里的"热点"
- top3 里至少 1 个来自存量池×时事的组合（如果存量池非空）
- 发布后数据回填 memory（post_history），让评分锚点随真实数据迭代
